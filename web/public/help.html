Particle Simulator

Use this guide to load, run, edit, and create particle simulation scenarios from the browser interface.

Quick Overview

Particle Simulator lets you create and run 2D particle scenes in the browser. A scene can contain many particles, forces such as gravity or wind, boundaries, and obstacles. The simulator shows how those particles move and collide over time.

A scenario is the setup for a simulation. It describes what particles exist, where they start, how they move, what forces affect them, and what area they are allowed to move inside. Scenarios can be loaded from bundled examples, uploaded as JSON files, edited in the app, downloaded, and run.

Main Page Controls

  • Play / Pause: starts or pauses the current simulation.
  • Stop: resets the current simulation back to the start and leaves it paused.
  • Speed slider: changes playback speed in fixed steps.
  • Particles: shows the number of particles in the running scenario.
  • Sim Time: shows how much simulation time has elapsed.
  • Scenario: shows the selected or custom scenario.

Builder Buttons

  • Choose Scenario: opens the scenario picker for bundled examples.
  • Load: loads the selected bundled scenario into the builder.
  • Upload: imports a scenario JSON file from your computer.
  • Download: saves the current scenario as a JSON file.
  • Run: starts a simulation using the current scenario.
  • Additional Settings: opens the full scenario editor.

Typical Workflow

  1. Click Choose Scenario and select a bundled scenario, or click Upload.
  2. Click Load if you chose a bundled scenario and want to inspect or edit it first.
  3. Click Run to start the simulation.
  4. Use Play / Pause, Stop, and the Speed slider to control playback.
  5. Use Additional Settings to customize the scenario.
  6. Use Download to save a scenario you want to keep.

Creating a Custom Scenario

Use Additional Settings to create or edit a custom scenario. The settings page is organized into sections, and each section controls a different part of the simulation.

Window Settings

Window settings describe how the simulation appears. You can configure width, height, title, background color, and target FPS. These settings affect presentation and usually do not change particle physics.

Runtime Settings

Runtime settings are derived from target FPS. The app chooses the fixed physics timestep, repeatable seed, and collision iterations automatically.

If a simulation feels heavy, try lowering particle count or target FPS.

Forces

Forces push or pull particles during the simulation. Gravity applies constant acceleration, drag slows particles down, wind pushes in a chosen direction, and radial forces pull particles toward or push particles away from a point.

Particle Types

Particle types define reusable particle properties such as radius, mass, restitution, color, and initial velocity. Use multiple particle types when the same scenario needs different kinds of particles.

Spawn Groups

Spawn groups create actual particles from particle types. A spawn group controls which particle type to use, how many particles to create, where they can appear, and what range of starting velocities they can have. Radius, mass, restitution, color, and initial velocity always come from the referenced particle type. Create another particle type when a group needs different physical or visual properties.

Geometry

Geometry defines the physical space of the simulation. Bounds act like the outer walls. Obstacles are fixed shapes inside the simulation area that particles can collide with.

Obstacles

Obstacles can be rectangles or circles. Rectangle obstacles use position and size. Circle obstacles use center and radius. Restitution controls how bouncy collisions with the obstacle are.

JSON Preview

The JSON preview shows the scenario data that the app will run or download. You do not need to edit JSON manually, but it is useful for inspecting, sharing, or debugging a scenario.

Scenario Building Tips

  • Start from a bundled scenario, then modify it.
  • Keep particle counts low while experimenting, then increase them later.
  • Make sure spawn positions are inside the simulation bounds.
  • Use drag to calm down fast or chaotic simulations.
  • Download your scenario after making changes you want to keep.
